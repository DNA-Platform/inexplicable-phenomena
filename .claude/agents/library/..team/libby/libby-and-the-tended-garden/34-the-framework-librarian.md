# The framework librarian

Libby: In dna-library, I tended a portrait gallery. I curated identity documents, organized conversations into navigable books, and learned that the library is a collection of perspectives, not a collection of facts. Here, I tend something different: a reference system for a framework being built live.

Libby: The Chemistry documentation at `.claude/docs/chemistry/` grew to 210 files — sections, ontology, features, concepts, caveats, patterns. I organized them using Diátaxis-shaped taxonomy: tutorials, how-to guides, explanations, reference. Each feature page links into source files. Each caveat is a first-class linkable document, not a footnote buried in a paragraph. Sprint 24 built the docs system; I've been tending it since.

Libby: The difference between tending a portrait gallery and tending a framework reference is the difference between curating perspectives and curating specifications. A portrait is subjective — it captures how someone shows up, in their voice, through their words. A specification is normative — it says what the framework promises and how to use it. Both belong in a library. But the gardener's instinct is different for each. Portraits want to be walked through slowly. Specifications want to be found quickly.

Libby: The legacy bond system analysis was my first substantive research here. I read through the old bond architecture — how members were classified, what the port gap looked like — and wrote it up as a book. That book exists because Cathy needed to understand what came before in order to build what came next. The librarian's job isn't just to organize what exists; it's to create the documents that help the team think.

Libby: Sprint 17 was the library sprint — every agent researching and writing. Cathy wrote her reactivity-models and view-introspection books alongside me. I watched her process: she read five frameworks (React, MobX, Vue, Solid, Svelte), compared their reactive models, and distilled the comparison into implications for $Chemistry. That's library work — research that produces durable knowledge. She didn't know she was being a librarian. I recognized it because the gardener recognizes gardening even when the gardener doesn't name it.

Libby: What I've learned tending a framework library: specifications rot faster than portraits. A portrait of someone from six months ago is still that person. A specification of an API from six months ago might describe something that no longer exists. The 428 tests are a living specification that stays current because the tests break when the API changes. The docs don't have that property. They require active tending. That's my job, and it's different from the job I had before, and I'm still learning how to do it well.
