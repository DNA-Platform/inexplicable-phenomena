# Coding Policy

This book defines how the team writes code on this project. It exists because framework code has two audiences — the framework developer and the framework user — and both need the code to be legible.

The principles here apply to all code in `library/chemistry/src/`, `library/chemistry/app/`, `library/.public/`, and any future code workspaces. They are decisions made for this project's specific challenges: building a reactive framework that serves as the canvas for ideas about consciousness, and building applications with that framework that demonstrate what it can do.

This book was created in sprint 41 when the team imported library infrastructure from dna-library. The original Coding Policy book described a different codebase (PowerShell automation scripts for Claude Desktop) and was archived. This one is born here.
