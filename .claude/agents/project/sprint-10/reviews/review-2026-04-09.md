## Understanding

The chemistry directory contains 9 source files implementing a layered component framework on React.

**particle.tsx (132 lines)** — The irreducible unit. Identity (CID, symbol, type), lifecycle (`next(phase)`), prop application, and the `$bond$` extension point. The free function `use()` hoists a particle into a React FC with lifecycle hooks and view-diffing cache.

**diff.ts (34 lines)** — ReactNode tree comparator. `diff(a, b)` returns true when trees differ. Used by `use()` to return cached references when views are unchanged.

**chemical.ts (172 lines)** — Extends particle with templates, bonds, orchestration, parent/catalyst hierarchy. Overrides `$bond$` with molecule/orchestrator pipeline. The free function `bind()` creates prototypal child instances. Bottom-of-file deferred wiring handles circular dependencies.

**molecule.ts (307 lines)** — `$Reflection` (property annotations), `$Molecule` (structural bond collection), `$Bond` (property metadata), `$Bonding` (method wrapper that triggers `$update$`), `$Parent` (parent property bond).

**reaction.ts (34 lines)** — Stripped to identity tracking. Chemical registration, parent-child hierarchy, static `find()` lookup.

**atom.ts (95 lines)** — `$Atom` (template-returning chemical) and `$Persistent` (formation lifecycle with localStorage).

**orchestrator.ts (~600 lines)** — Bond orchestrator, orchestration context, param validation. Processes JSX children into binding constructor arguments. Largely untouched from the v1 lift.

**wrappers.ts (~50 lines)** — `$Function$`, `$Html$`, `$Include`, `$Exclude`. React FC and HTML element wrappers.

**helpers.ts (~220 lines)** — `$Represent`, `$symbolize`, `$literalize`, `$promise`, `$use`, `$wrap`, `$check`, `$`, `$is`, `$await`.

## Findings

### 1. `$Bonding` still uses `Object.defineProperty` — inconsistency (Now)
- **File:** molecule.ts:274
- **Anxiety:** Naming that misleads about behavior
- **Finding:** We said "no Object.defineProperty for reactive property interception" — but `$Bonding.form()` STILL calls `Object.defineProperty` to wrap methods. The property itself is replaced with a getter that returns a wrapped function. This IS property replacement, just for methods instead of data. The inconsistency is: data bonds are pure metadata, method bonds replace the property. The developer doesn't know which properties are replaced and which aren't.
- **Recommendation:** This is the RIGHT behavior for now — methods need wrapping to trigger `$update$`. But we should acknowledge it: the rule is "no interception for data properties, wrapping for methods." Document it. And consider whether there's a cleaner way to wrap methods without `Object.defineProperty` (e.g., Proxy, or wrapping at call-time in the view).

### 2. `use()` closes over a single particle instance — no instance forking (Soon)
- **File:** particle.tsx:102-129
- **Anxiety:** Abstractions that don't compose
- **Finding:** `use(particle)` captures the particle in a closure. Every React render of this component mutates the SAME particle via `$apply`. If the component is rendered twice (e.g., `<Counter /><Counter />`), both renderings share the same particle — same `count`, same `$label`. There's no instance forking. This is correct for `$Chemical` (which has `bind()` for creating child instances), but for direct `use()` on a particle, it's a footgun.
- **Recommendation:** Document that `use()` creates a 1:1 particle-to-component binding. For multiple instances, create multiple particles. For chemicals, use `bind()` or the `Component` getter which manages templates.

### 3. Deferred wiring at bottom of chemical.ts — fragile (Soon)
- **File:** chemical.ts:158-172
- **Anxiety:** Hidden dependencies not visible in the import graph
- **Finding:** `$setChemicalClass`, `_setParamValidation`, `$setRuntime`, and a `queueMicrotask` with dynamic imports. This is circular-dependency workaround machinery. It works in vitest but is order-dependent and non-obvious. If module loading order changes, the wiring might not complete before first use. The `queueMicrotask` is especially fragile — it relies on microtask timing relative to module initialization.
- **Recommendation:** Track as tech debt. The diffing refactor simplified the dependency graph (no more reaction→state→bond cycles). Revisit whether the remaining circular deps can be broken structurally.

### 4. orchestrator.ts is untouched v1 code (~600 lines) — stale (Eventually)
- **File:** orchestrator.ts
- **Anxiety:** Type system fighting the runtime
- **Finding:** The orchestrator was lifted verbatim from the archive and hasn't been updated for the new architecture. It still references `$molecule$` reactivation inside its `render()` method (which is now called from `$bond$`, not from a `$Component$`). The `augmentView()` method rewrites the React element tree — its purpose hasn't been validated post-refactor. The `$ParamValidation` class uses duck-typing checks for `$Chemical` descendants because it can't import the class. The runtime wiring (`$setRuntime`) patches in 7 external references.
- **Recommendation:** The orchestrator needs its own deep read and refactor. It's the largest file and the least understood. Not urgent — it works — but it's the next target for simplification.

### 5. `$Persistent` calls `formula()` and `read()` which use `$symbolize` — dormant serialization (Intentional)
- **File:** atom.ts:64-87, molecule.ts:104-127
- **Anxiety:** Dead paths that can't be reached
- **Finding:** Doug said remove all USE of serialization from rendering paths. `$Persistent.reform()` and `reflect()` still call `$Molecule.read()` and `formula()`, which call `$symbolize` and `$literalize`. These are dormant — nobody instantiates `$Persistent`. The methods exist for future use.
- **Recommendation:** Intentional. Leave them. But nobody should call `formula()` or `read()` from a rendering path. The integration test proves rendering works without them.

### 6. diff.ts doesn't check for `ref` — minor (Eventually)
- **File:** diff.ts:14-17
- **Anxiety:** Breaking React's rules
- **Finding:** The comparator checks `type` and `key` but not `ref`. If two elements have different refs, `diff()` says they're the same (because refs aren't in `props`... actually refs ARE in the element object but React handles them specially). In practice, refs are rare in $Chemistry views (the object model replaces most ref use cases).
- **Recommendation:** Low priority. If refs become relevant, add the check.

### 7. No test for the actual re-render cycle triggered by `$Bonding` — gap (Now)
- **File:** tests/
- **Anxiety:** Missing tests for load-bearing behavior
- **Finding:** The integration test proves initial render and one re-render (`addPages(307)`). But we don't test: multiple rapid re-renders, async methods triggering re-renders on promise resolution, the view cache actually returning the cached reference (proving React skips work), or the `$phase$ !== 'setup'` guard preventing render-time re-triggers.
- **Recommendation:** Write focused tests for the re-render mechanism. These are the tests that prove diffing works under stress.

### 8. `$Reaction` might not need to exist — structural question (Eventually)
- **File:** reaction.ts (34 lines)
- **Anxiety:** Premature generalization
- **Finding:** `$Reaction` is now 34 lines: a constructor, `add()`, `destroy()`, and a static `find()` map. The chemical constructs it, uses `add()` for parent-child registration, and `destroy()` for cleanup. The static map could live on `$Chemical` itself. The reaction system hierarchy (child reactions sharing a system root) is unused in the diffing model — the catalyst system references it but doesn't leverage it for re-rendering.
- **Recommendation:** Consider merging `$Reaction` into `$Chemical` or making it a simple registry. The class adds indirection without earning its boundary.

## Strengths

**Cathy:** The particle is clean. 99 lines of class, 30 lines of `use()`. Identity, lifecycle, view, extension point. Every line earns its existence. The `next()` method is elegant — 8 lines that handle setup/mount/effect/unmount ordering with smart resolution. The `$bond$` extension point is the single seam where chemical plugs in its complexity — one method, one override. That's faithful to the particle/chemical layering.

**Cathy:** The diff is honest. 34 lines. It does one thing. It halts on first difference. It doesn't try to be clever. The cleverness is in the architecture — that we CAN diff because views are pure. The diff itself is just a tree walk.

**Cathy:** The `$Chemical.$bond$` override is three lines:
```typescript
this[$molecule$].reactivate();
this[$orchestrator$].bond({ children: this[$children$] });
this[$molecule$].reactivate();
```
That's the entire orchestration pipeline plugged into the particle's render path. Molecule, orchestrator, molecule again. Dense, correct, minimal.

**Arthur:** The dependency graph is dramatically simpler than before. Particle → nothing (except diff). Chemical → particle, molecule, reaction, orchestrator. No circular runtime deps in the core path. The deferred wiring is confined to chemical.ts's bottom and only affects wrapper/helper registration. The build compiles without `@ts-nocheck` anywhere.

**Arthur:** The line count tells the story. Before sprints 8-10: molecule.ts (681) + reaction.ts (245) + component.ts (104) = 1,030 lines for the rendering machinery. After: molecule.ts (307) + reaction.ts (34) + diff.ts (34) = 375 lines. component.ts deleted. The archive (5,382 lines) deleted. The framework is lighter and the code it does have is load-bearing.

## Summary

**Cathy:** The particle/chemical layering is faithful. The view is pure. The diff works. The integration test proves the reactive loop. I'm proud of particle.tsx and diff.ts. I'm not proud of the orchestrator — it's a v1 transplant we haven't made our own. And the test coverage for the re-render mechanism is thin. We prove it works once. We don't prove it works under stress.

**Arthur:** The architecture is sound but the edges are rough. The deferred wiring is pragmatic debt. The orchestrator is borrowed code. The test gaps are real. What we have is a solid core — particle, chemical, diff — with a periphery that needs the same care the core received.

**Arthur:** The single most important thing: write the re-render stress tests. The diff is the framework's thesis. If it breaks under rapid mutation, async resolution, or complex view trees, the thesis fails. Prove it doesn't.
