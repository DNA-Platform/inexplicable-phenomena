# Ontology of $Chemistry

What `$Chemistry` *is*. This is the static spine of the framework: the entities that exist, the relationships between them, the concepts they instantiate, and the corners that demand explicit teaching.

The ontology is one of three navigation axes. The other two are the [epistemology] (how we know it works) and the [topical progression] (introductory to sophisticated). Where the topical progression is a *route*, the ontology is a *map* — flat, indexed, no narrative order.

## Sub-spines

- [entities] — the things that exist: particles, chemicals, atoms, bonds, reactions, molecules, syntheses, the `$()` callable.
- [relationships] — how the entities connect: derivative-and-template, parent-and-catalyst, reactive-prop-as-react-prop, chemical-as-react-component.
- [concepts] — the abstractions the entities instantiate: state, reactivity, composition, lifecycle, particularization, lexical scoping, render filtering.
- [surprising] — the corners that demand explicit teaching. The bond constructor lives here. So does `instanceof Error` working through a particularized carrier.

## Reading order

There is no required reading order at this layer; that's the [topical progression]'s job. If you arrived here with a specific question — "what is a `$Chemical`?" — start in [entities]. If you arrived to understand a counterintuitive result — "why does `instanceof Error` work on my `new $Particle(error)`?" — start in [surprising].
